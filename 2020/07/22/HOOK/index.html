WHAT IS HOOK

在计算机中，当程序执行时，指令流也像水流一样，只要在合适的位置下钩子，就可以对程序的运行流程进行监控、拦截。HOOK的关键就是通过一定手段埋下“钩子”，“钩”住我们关心的重要流程，然后根据对需要对执行过程进行干预。

Address Hook

Address Hook是指通过修改数据进行hook的方法。这里的’‘数据“通常指一些函数的地址（也可能是偏移量）

它们通常存放在各类表或结构中，或者某个指定的地址处，抑或特殊的寄存器中。他们的共同点是，在某个时刻会成为程序运行的eip。因此只要把这些地址替换成我们的Detour函数地址，就可以轻松拿到程序的控制权了。

各类表中的地址

PE的IAT

IAT模块具体指某个PE模块的IAT，所以它的作用范围只针对被HOOK的模块，且必须在以静态链接的方式调用API时才会被HOOK，在使用LoadLibrary或GetProcAddress进行动态调用时不受影响（在其他模块调用被HOOK的API时也没有影响）。

IAT的全称是Import Address Table。

IAT表是执行程序或者dll为了实现动态加载和重定位函数地址，用到的一个导入函数地址表。这里面记录了每个导入函数的名字和所在的dll名称，在pe加载的时候系统会加载这些dll到用户的地址空间然后把函数地址覆盖这个表里的函数地址，然后重构所有用到这个表的代码，让其调用直接指向实际函数地址（PE是否覆盖不确定，驱动会这么做），PE的IAT表会留在内存，驱动的就丢弃了。

PE的EAT

HOOK后，所有试图通过输出表获取函数地址的行为都会受到影响（主程序及其输入表中所有模块的IAT在程序初始化的时候就已经被填充了，因此，之后通过IAT进行调用不受EAT HOOK的影响）。EAT并不直接起作用，它只能影响Hook之后对该函数的地址的获取（例如IAT拦截不到的LoadLibrary或GetProcAddress调用方式）或者后续加载的模块的IAT。

Export Address Table

在windows系统中，“库”是为了方便其他程序调用而集中包含相关的函数的文件(dll,sys).win32 API是最具有代表性的库。
EAT是一种核心机制，它使不同的应用程序可以调用库文件中提供的函数。也就是说，只有通过EAT表才能准确的求得从相应的库中导出函数的起始地址。

EAT中存放的不是函数地址，而是函数的偏移地址，使用时需要加上模块基址

user32.dll的回调函数表

被作为实参传入另一函数，并在该外部函数内被调用，用以来完成某些任务的函数，称为回调函数。

IDT

中断描述符表（Interrupt Descriptor Table，IDT）将每个异常或中断向量分别与它们的处理过程联系起来。

IDT的表基址存放在idtl寄存器中，每个中断项的中断处理例程称为ISR(Interrupt Service Routine)

SSDT和Shadow SSDT

SSDT(System Srevice Descriptor Table)，系统服务描述符表

SSDT：主要处理 Kernel32.dll中的系统调用，如openProcess,ReadFile等，主要在ntoskrnl.exe中实现（微软有给出 ntoskrnl源代码）

ShadowSSDT: 1.主要处理，user32.dll,GDI32.dll中调用的函数，如postMessage,SendMessage,FindWindow,主要在win32k.sys中实现.（微软未给出win32k代码）

c++在使用虚函数和重载时，使用了一个地址表来保存寻函数的地址，称为虚函数表（Virtual Function Table，VFT）.一旦子类重载了某个函数，编译器就会用子类中该函数的地址替换虚函数表中对应的函数地址。重载就是用子类的方法去覆盖父类的方法，也就是地址替换（实际上在HOOK时也是这么做的）。

COM接口的功能函数表

com接口同样存在vftable，与c++中的vftable完全一样。替换vftable中的地址，就可以实现对COM接口的HOOK。

开发人员通常只会根据需要替换表中的个别地址来进行HOOK，但如果直接替换这个地址表的基址就会可以在不修改原表内容的情况下对全部项目进行HOOK了（实际使一种大规模的Address Hook）。

特殊寄存器中的地址

Windows使用MST寄存器组中IA32_SYSENTER_EIP的值作为内核调用的入口，当在ntdll中调用汇编指令stsenter进入内核时，CPU会首先执行到这里。

因此，通过修改MSR寄存器中保存的快速调用入口函数地址，同样可以进行Hook。

Inline Hook

直接修改指令的Hook，一般使用jmp,call,retn之类的转移指令。

jmp xxxxx（5字节）

直接跳转到某地址

push xxxxxxx/retn（6字节）

通过压栈返回实现跳转

mov eax,xxxxxxx/jmp eax（7字节）

先将转移地址放入寄存器，再实现跳转

选用eax寄存器的原因是因为eax寄存器通常用于保存函数的返回值，在函数入口修改它不会影响函数执行结果
